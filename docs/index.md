      ## The thesis

      Most AI content stops at *"the model predicts…"*

      This repository keeps going — through *"the email fires,"* *"the agent sees this script,"* and *"measurement closes the loop."*

      The activation layer is where intelligence becomes outcomes. That gap is where most enterprise AI value is lost, and where this work focuses.

      ---

      ## Repository structure

      Each artifact is one of four types, declared in frontmatter:

      | Type | Purpose |
      |---|---|
      | **Reference architecture** | Diagram-led, vendor-agnostic at the framework level |
      | **Decision record** | Short, sharp — captures one architectural decision and its rationale |
      | **Case pattern** | Sanitized industry pattern drawn from delivery experience |
      | **POV** | Position piece — argued, not just described |

      Filename convention: `YYYY-MM-DD_<vertical>_<pattern-type>_<slug>.md`
